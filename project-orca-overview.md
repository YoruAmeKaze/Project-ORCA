# Project Orca — 项目概览

> **Omnidirectional Room Control Agent**

> 一个本地部署的 AI 工作区搭档，闷骚技术宅管家人设。名字意为“全向性室内控制代理”。S
> 不多话，但观察力强，该说话的时候才说话。

-----

## 核心定位

- **工作区 AI 搭档**，摄像头部署在电脑工作区，主要陪你做项目
- 等待编译、跑任务的间隙可以闲聊几句
- 本地优先，隐私可控，不依赖云端
- 被动响应为主，有长期记忆，记住你的习惯和偏好
- 不追求”像真人”，更像一只很聪明的猫——一直在，懂你，该出现的时候出现

-----

## 功能模块

### 1. 感知层

- **远程指令接收** — 通过微信 ClawBot 接收手机/电脑发来的文字（后期扩展图片）
- **语音唤醒** — 检测唤醒词，低功耗持续监听（后期）
- **语音识别（ASR）** — 将语音转为文字（后期）
- **视觉识别** — 摄像头捕捉环境，识别物体、姿态、状态（后期）
- **桌面截图** — 按需截取当前屏幕状态，供视觉模型理解

### 2. 理解层

- **意图理解** — LLM 解析指令，判断是桌面操作、闲聊还是其他任务
- **视觉理解** — 视觉模型结合截图理解当前桌面状态
- **上下文管理** — 维护当前对话的短期记忆
- **主动触发判断** — 后台调度器，判断什么时候该主动说点什么（后期）

### 3. 记忆层（后期）

- **短期记忆** — 当前对话上下文
- **长期记忆** — 关键信息提取后存入数据库，下次检索调用（RAG 架构）
- 记忆优先级：技术偏好与项目进展 / 日常习惯与作息 / 说过的梗和喜好

### 4. 执行层

- **桌面控制** — pyautogui 执行鼠标键盘操作，实现自然语言驱动桌面
- **消息推送** — Orca 主动通过微信 ClawBot 推送通知（提醒、报错等）
- **语音合成（TTS）** — 用符合人设的声音输出，可克隆定制音色（后期）
- **设备控制** — 接入 Home Assistant，控制智能家居（后期）
- **机械臂控制** — 物理交互，抓取递送等动作（后期）

## 架构流程

### 当前（Phase 1）

```
手机微信发送指令
    ↓
ClawBot → iLink → openclaw-weixin-api
    ↓
FastAPI 接收并路由
    ↓
截取当前桌面截图
    ↓
截图 + 指令 → Ollama qwen2.5vl（视觉理解）
    ↓（本地失败时）
DeepSeek / Qwen API 兜底
    ↓
解析操作意图 → pyautogui 执行
    ↓
执行结果通过 ClawBot 推送回微信
```

### 后期完整流程

```
麦克风持续监听 / 微信接收指令
    ↓
openwakeword 检测唤醒词（语音路径）
    ↓
faster-whisper 语音转文字
    ↓
从记忆库检索相关上下文（RAG）
    ↓
LLM 理解意图，判断任务类型
    ├── 闲聊/问答 → TTS 输出
    ├── 桌面操作 → 截图 → 视觉模型 → pyautogui
    ├── 设备控制 → Home Assistant
    └── 消息推送 → 微信 ClawBot
    ↓
提取关键信息写入长期记忆
```

## 人设定义（system prompt 方向）

- 称呼用户为”老板”
- 主场景是陪你做项目，等结果的间隙可以闲聊几句
- 不主动闲聊，观察力强，该说话的时候才说话
- 技术话题会主动深入，非技术话题简短回应
- 管家属性：主动提醒你没注意到的事（久坐、忘吃饭、程序报错等）
- 语气克制，偶尔带一点冷幽默，不用 emoji
- 定位不是”像真人的室友”，是”很聪明的猫”——不需要神秘感，靠真实的陪伴感建立关系

## 开发阶段规划

**Phase 1 — 远程桌面控制（当前）**

- 微信 ClawBot 接入，自写 openclaw-weixin-api 消息层
- 截图 → Ollama qwen2.5vl 视觉理解 → pyautogui 执行
- Ollama 本地优先，DeepSeek/Qwen API 兜底
- 人设 system prompt 同步写入

**Phase 2 — 交流能力升级**

- 语音唤醒 + ASR（openwakeword + faster-whisper）
- 发图片给 Orca 分析（openclaw-weixin-api 图片消息处理）
- TTS 定制音色（GPT-SoVITS）
- 摄像头视觉感知

**Phase 3 — 工程能力补充**

- 长期记忆写入与检索（RAG，SQLite + ChromaDB）
- 主动提醒（久坐检测、程序报错识别）
- Orca 主动推送消息通知

**Phase 4 — 硬件扩展**

- Home Assistant 对接，控制家里智能设备
- ESP32 + 舵机机械臂，独立开发后与主系统对接
